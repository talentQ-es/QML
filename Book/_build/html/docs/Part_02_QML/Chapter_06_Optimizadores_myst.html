6. Optimizadores#

Tal y como se menciona en el notebook de 1. IntroducciÃ³n, el aprendizaje automÃ¡tico tinene como objetivo aprender de las observaciones existentes y hacer uso de ese aprendizaje para predecir nuevas observaciones o determinar el resultado de nuevas entradas. Para ello, es necesario un proceso iterativo en el que se busca minimizar el error cometido, este proceso se conoce como el entrenamiento del modelo [3].

En objetivo del entrenamiento es lograr el menor error posible, para ello se evalua la funciÃ³n de coste conociendo asÃ­ la calidad de las predicciones del modelo, en otras palabras conoceremos como de buena es una soluciÃ³n. El proceso irÃ¡ avanzando, mejorando la precisiÃ³n en la predicciÃ³n, hasta cumplir con el criterio de parada fijado. La tarea de minimizar las funciones de coste no es sencilla y existen diversas tÃ©cnicas para abordarla, algunas de ellas se muestran a continuaciÃ³n.

6.1. Descenso por gradiente#

El descenso por gradiente (gradient descent) es un algoritmo de optimizaciÃ³n utilizado para realizar el entrenamiento de modelos de Machine Learning y Deep learning que se basa en el concepto del gradiente para llevar a cabo la minimizaciÃ³n del error cometido [4]. Cabe destacar que para poder encontrar el mÃ­nimo de la funciÃ³n este mÃ©todo asume que se puede obtener el gradiente de la funciÃ³n a estudiar y que esta es contÃ­nua y diferenciable (no necesariamente en toda la funciÃ³n).

Para poder implementar este algoritmo de optimizaciÃ³n se debe definir la direcciÃ³n y la tasa de aprendizaje (tambiÃ©n conocido como el tamaÃ±o del paso). El primero de ellos nos indica en quÃ© direcciÃ³n seguir explorando el conjunto de soluciones mientras que el segundo especifica cuanto avanzar en la direcciÃ³n seleccionada [1].

El algoritmo parte desde un punto arbitrario \(x^{(0)}\) en el que se debe conocer el valor inicial de la funciÃ³n objetivo a minimizar. En cada paso \(k \geq 0\) se avanza de forma iterativa en la direcciÃ³n opuesta al gradiente, ya que se trabaja con problemas a minimizar. Cabe destacar que esto no supone un lÃ­mite a la hora de definir la cantidad de problemas que se pueden resolver con esta tÃ©cnica, ya que todos los problemas a maximizar se pueden transformar en problemas a minimizar mediante un cambio de signo. La busqueda de la soluciÃ³n utilizando el descenso por gradiente se puede resumir mediante la siguiente expresiÃ³n matemÃ¡tica.

\[ x^{(k+1)} = x^{(k)} - t_k \nabla f(x^{(k)})\]

Donde \(x^{(k+1)}\) corresponde al nuevo punto, \(x^{(k)}\) representa el punto actual en el que se encuentra el algoritmo, \(t_k\) hace referencia al tamaÃ±o del paso (tambiÃ©n se utiliza \(\alpha\) para representar este parÃ¡metos) y \(\nabla f(x^{(k)})\) corresponde al gradiente en el punto actual.

Tal y como se aprecia en la expresiÃ³n anterior, para encontrar el nuevo punto el algoritmo se avanza en el sentido opuesto al gradiente, esto se indica mediante el signo negativo.

A continuaciÃ³n, se adjunta el pseudo-cÃ³digo del descenso por gradiente.

Nota

La tasa de aprendizaje tambiÃ©n conocida como tamaÃ±o del paso, suele tomar un valor pequeÃ±o. Existen algunas variantes del algoritmo en las que se evalÃºa y se actualiza en funciÃ³n del comportamiento de la funciÃ³n de coste. Mediante estas estrategias se busca explorar el espacio de soluciones en las primeras iteraciones y explotar las buenas soluciones conforme avanzan las iteraciones [3].

Una tasa de aprendizaje alta conlleva pasos mÃ¡s grandes, por lo que puede que el algoritmo encuentre la soluciÃ³n antes, pero se corre el riesgo de sobrepasar el minimo y que el algoritmo no converja a una soluciÃ³n. El descenso del gradiente con una tasa baja, sin embargo, avanza muy despacio y necesita muchas iteraciones para encontrar una buena soluciÃ³n, resultando asÃ­ un proceso muy costoso.

6.1.1. Problemas del descenso por gradiente#

El algoritmo de descenso por gradiente se basa en que la funciÃ³n de coste es convexa, cuando esta condiciÃ³n se cumple el algoritmo de descenso por gradiente encuentra con facilidad el mÃ­nimo global. En problemas no convexos sin embargo, este algoritmo puede encontrarse con dificultades para llegar hasta el minimo global y quedarse atrapado en minimos locales.

Es importante recordar que cuando la pendiente de la funciÃ³n de coste (el gradiente de la funciÃ³n) es cercana a cero el modelo deja de aprender, dejando al algoritmo atrapado en dichos puntos. Cabe destacar que esta situaciÃ³n se puede producir tambiÃ©n en puntos silla (tambiÃ©n llamados puntos de inflexiÃ³n) y mÃ­nimos locales, por lo que en estos casos hacer uso del descenso por gradiente no asegura el obtener la soluciÃ³n Ã³ptima al problema.

Se detectan otras dos dificultades al utilizar el descenso por gradiente para ajustar los parÃ¡metros de las redes neuronales.

  • Desvanecimiento del gradiente (gradientes fuga), esto ocurre cuando el gradiente es demasiado pequeÃ±o en la retropropagaciÃ³n en las redes neuronales. El gradiente se reduce lo que provoca que las capas intermedias de la red aprendan muy lentamente, cuando esto sucede los parÃ¡metros se actualizan hasta que toman el valor nulo. Cuando esto ocurre el algoritmo que se esta entrenando deja de aprender. [4]

  • El problema de gradientes explosivos es justamente el efecto opuesto al anterior, es decir, cuando el gradiente es demasiado grande. En estos casos se crea un modelo inestable, los pesos de la red neuronal crecen demasiado hasta que pasan a tomar el valor NaN. [4]

def create_circuit(w,x, nqubits= 2):
    
    c= Circuit(nqubits=nqubits)
    
    # ENCODING:

    c.add(gates.RY(q=0, theta = w[0]*x[0]+w[4]))
    c.add(gates.RY(q=1, theta = w[1]*x[1]+w[5]))
    
    c.add(gates.RZ(q=0, theta = w[2]*x[2]+w[6]))
    c.add(gates.RZ(q=1, theta = w[3]*x[3]+w[7]))
    
    # CIRCUITO VARIACIONAL:
    
    # LAYER 1
    c.add(gates.RY(q=0, theta = w[4]))
    c.add(gates.RY(q=1, theta = w[5]))
     
    c.add(gates.CZ(0,1))
    
    c.add(gates.RY(q=0, theta = w[6]))
    c.add(gates.RY(q=1, theta = w[7]))
    
    # LAYER 2
    c.add(gates.RY(q=0, theta = w[8]))
    c.add(gates.RY(q=1, theta = w[9]))
    
    c.add(gates.CZ(0,1))
    
    c.add(gates.RY(q=0, theta = w[10]))
    c.add(gates.RY(q=1, theta = w[11]))
    
    # LAYER 3
    c.add(gates.RY(q=0, theta = w[12]))
    c.add(gates.RY(q=1, theta = w[13]))

    c.add(gates.CZ(0,1))
    
    c.add(gates.RY(q=0, theta = w[14]))
    c.add(gates.RY(q=1, theta = w[15]))
    
    # ÃšLTIMA ROTACIÃ“N EN Y:
    c.add(gates.RY(q=0, theta = w[16]))
    c.add(gates.RY(q=1, theta = w[17]))
    
    # MEDIMOS EL QUBIT 0:
    c.add(gates.M(0))
    
    return c

La siguente funciÃ³n calcula el rendimiento del modelo, para ello se utilizan dos mÃ©tricas:

DefiniciÃ³n: Error cuadrÃ¡tico medio

Mide el promedio del error al cuadrado, donde el error se considera la diferencia entre la predicciÃ³n que realiza el modelo y la etiqueta almacenada en el dataset. La formula con la que se calcula este valor es la siguiente:

\[ MSE = \frac{1}{N} \sum_{i=1}^N (y_i-\hat y_i)^2 \]

DefiniciÃ³n: Accuracy

Se trata de una mÃ©trica que muestra fracciÃ³n de ejemplos que se han clasificado correctamente, es decir, que el modelo ha predicho la clase a la que pertenece.

\[ \text{Accuracy} = \frac{\text{Number of correct pedictions}}{\text{Total number of predictions}} \]
def sqloss_acc(labels, predictions):
    sqloss = 0
    acc = 0
    # Recorremos todas las instancias comparando el label conocido con la predicciÃ³n que ha realizado el modelo.
    for label, prediction in zip(labels, predictions):
        
        # Modificamos el valor de loss
        sqloss = sqloss + (label - prediction)**2
        
        #Vemos si el label y la predicciÃ³n coincide para calcular el accuracy
        if np.sign(label)==np.sign(prediction):
            acc += 1
    # dividir entre la cantidad de datos de entrada para obtener un valor entre 0 y 1.
    sqloss = sqloss / len(labels)
    acc = acc / len(labels)
    
    print('Loss: ', sqloss.numpy(), 'Acc: ', acc)
    return sqloss, acc

#  En esta funciÃ³n se llama a la funciÃ³n de crear el circuito con los parÃ¡metros necesarios.
def Classifier_circuit(theta,data,nqubits=2):
    circ = create_circuit(theta,data, nqubits)
    return circ

La funciÃ³n hamiltonian varÃ­a en funciÃ³n del nÃºmero de qubits con los que se este trabajando, en este caso nos interesa realizar una mediciÃ³n Z en el qubit 0 por lo que realizamos el producto Z en el qubit 0 y la identidad para el resto de qubits. Haremos uso de estÃ¡ funciÃ³n para calcular el valor esperado del circuito y nos facilitarÃ¡ el cÃ¡lculo del gradiente que necesitaremos mÃ¡s adelante.

# Definir el observable:
def hamiltonian():
    Obj = np.prod([ Z(0), I(1)])
    h = hamiltonians.SymbolicHamiltonian(Obj)
    return h

La funciÃ³n Predictions es la encargada de ejecutar el circuito y obtener el valor esperado. Tiene como parÃ¡metros de entrada el nÃºmero de qubits, el nÃºmero de shots, los pesos (son los parÃ¡metros que deseamos optimizar y corresponden a las rotaciones de las puertas del circuito) y una instancia x del dataset (unicamente las variables de entrada, excluimos el valor de la etiqueta).

def Predictions(data, theta, nqubits, nshots=10000):

    c = Classifier_circuit(theta,data,nqubits)
    h = hamiltonian()
    expected_value = h.expectation(c.execute().state())
    
    return expected_value

En la funciÃ³n de coste realizamos las predicciones para los datos de entrada y calculamos el rendimiento del modelo.

def Cost_function(theta, data, Y, nqubits, nshots=10000):
    predictions = [Predictions(x, theta,nqubits) for x in data]
    return sqloss_acc(Y, predictions)[0]

6.2. Momentum#

Como se ha comentado en apartados anteriores, el algoritmo de descenso por gradiente puede encontrarse con ciertas dificultades que empeoren considerablemente su funcionamiento. Esto principalmente se debe a que el resultado obtenido tiene una gran dependencia de la tasa de aprendizaje y del gradiente del paso actual, por lo que al no tener en cuenta la trayectoria de soluciones exploradas se puede quedar atrapado en puntos que aparentan ser una buena soluciÃ³n (por ejemplo mÃ­nimos locales y puntos silla). Para intentar solventar este tipo de dificultades se implementa el concepto de Momentum.

6.2.1. Algoritmo#

Este algoritmo es una extensiÃ³n de la tÃ©cnica anterior, por lo que la estrategÃ­a a seguir es muy similar, la variaciÃ³n que sufre esta tÃ©cnica es que en la exploraciÃ³n de soluciones se tiene en cuenta las soluciones ya exploradas. Para integrar la inercia se puede hacer uso del concepto estadÃ­stico de media mÃ³vil sobre los gradientes pasados. En las regiones donde la pendiente es alta, las actualizaciones serÃ¡n significativas, de manera que en cierto modo, se gana impulso al tomar una media mÃ³vil sobre estos gradientes.

Utilizando esta tÃ©cnica se consigue introducir la inercia. Sin embargo, utilizando este concepto estadÃ­stico todos los gradientes tienen la misma ponderaciÃ³n, en otras palabras, se les otorga la misma importancia a todos los gradientes independientemente de la lejanÃ­a del momento actual. Es por esto que se utiliza un promedio ponderado, de manera que los gradientes mÃ¡s recientes tengan mayor influencia.

El algoritmo no sufre variaciones drÃ¡sticas, unicamente se debe tener en cuenta la variaciÃ³n a la hora de calcular el gradiente. En el descenso por gradiente con momentum el valor del siguiente punto se calcula mediante la siguiente expresiÃ³n:

\[ x^{(k+1)} = x^{(k)} - a^{(k+1)}\]

donde el tÃ©rmino \(a^{(k+1)}\) se actualiza mediante la siguiente expresiÃ³n \( a^{(k+1)} = \beta a^{(k)} + t_k \nabla f(x^{(k)})\) donde \(t_k\) corresponde al tamaÃ±o de paso y \(\beta\) corresponde al momentum, este parÃ¡metro debe cumplir que \( \beta \in [0,1]\) [7].

6.2.2. Como seleccionar \(\beta\)#

Al introducir el momentum se debe ajustar un nuevo parÃ¡metro, este corresponde al peso que le otorgamos a los gradientes anteriores y se denomina \(\beta\). Si el valor de \(\beta\) es pequeÃ±o el gradiante se reduce rÃ¡pidamente pudiendo crear problemas de gradiente desvaneciente. Si se le asigna un valor demasiado grande, sin embargo, este no se reducirÃ¡n tan rÃ¡pido por lo que influirÃ¡n mÃ¡s gradientes en la actualizaciÃ³n, este segundo caso es el ideal. Cabe destacar que el valor mÃ¡s comÃºn para este parÃ¡metos es \(0.9\). [2]

6.3. Nesterov Momentum Optimizer (descenso por gradiente con Nertorv Momentum)#

Al igual que los dos mÃ©todos anteriores, este algoritmo de optimizaciÃ³n tambiÃ©n se basa en el gradiente. No obstante, cuenta con una mayor tasa de convergencia en ciertas situaciones. [9]

6.3.1. Algoritmo#

El optimizador Nesterov Momentum es similar al descenso por gradiente con momentum, pero este desplaza el punto actual mediante el tÃ©rmino de momentum al calcular el gradiente. De manera que la Ãºnica modificaciÃ³n respecto a la tÃ©cnica anterior es la expresiÃ³n para actualizar \(a^{(k+1)}\) que se sustituye por \(a^{(k+1)} = \beta a^{(k)} + t_k \nabla f(x^{(k)} - \beta a^{(k)})\).[8]

6.4 Adam#

Se trata de un mÃ©todo de optimizaciÃ³n estocÃ¡stica eficiente que unicamente requiere gradientes de primer orden con poco requerimiento de memoria. Este mÃ©todo calcula tasas de aprendizaje adaptativas individuales para diferentes parÃ¡metros a partir de las estimaciones del primer y segundo momento de los gradientes. Es por eso que el nombre de esta tÃ©cnica, Adam, proviene de adaptative moment estimation. [6]

6.4.1. Algoritmo#

A continuaciÃ³n se muestra el pseudo-cÃ³digo de este algoritmo.

Tal y como se muestra en la figura 5, el algoritmo actualiza las medias mÃ³viles exponenciales del gradiente (\(m_t\)) y del gradiente al cuadrado (\(v_t\)) donde los hiperparÃ¡metros \(\beta_1, \beta_2 \in [0,1)\) corresponden al parÃ¡metro de suavizado de las medias mÃ³viles. Como se aprecia en el pseudo-cÃ³digo anterior, los parÃ¡metros de suavizado se inicializan al 0, dejando que en las primeras iteraciones el valor sea cercano al valor nulo, especialmente cuando los valores de \(\beta\) son cercanos a 1 [6].

6.5 Adadelta#

Adadelta es una tÃ©cnica de optimizaciÃ³n estocÃ¡stica basada en el descenso por gradiente que permite aplicar el mÃ©todo de la tasa de aprendizaje por dimensiÃ³n. Se trata de una expansiÃ³n del mÃ©todo Adagrad que busca reducir la agresividad y el descenso monÃ³tono de la tasa de aprendizaje. Para ello en vez de hacer uso de todos los gradientes pasados, Ãºnicamente utilizan los contenidos en una ventana de tamaÃ±o fijo \(w\) [5].

6.5.1. Algoritmo#

La suma de los gradientes se define de manera recursiva, de manera que \(E[g^2]_t\) en el tiempo t depende de la media anterior y el gradiente actual, tal y como se indÃ­ca en la siguiente formula [11]: $\(E[g^2]_t = \gamma E[g^2]_{t-1} + (1-\gamma)g_t^2\)$

Normalmente el parÃ¡metro \(\gamma\) se fija a 0.9. Utilizando este mÃ©todo de optimizaciÃ³n los parÃ¡metros se acutalizan mediante las siguientes expresiones [11]: $\(\Delta\theta_t = -\frac{\eta}{\sqrt{E[g^2]_t+\epsilon}}g_t\)$

La principal ventaja del algoritmo Adadelta es que no es necesario definir el valor de la tasa de aprendizaje por defecto [5].

6.7. MÃ©todo de optimizaciÃ³n BFGS#

El algoritmo Broyden-Fletcher-Goldfarb-Shanno tambiÃ©n conocido como BFGS, es un mÃ©todo iterativo utilizado para resolver problemas no lineales sin restricciones. Al igual que el mÃ©todo Davidon-Fletcher-Powell, determina la direcciÃ³n de descenso precondicionando el gradiente con la informaciÃ³n de la curvatura de la funciÃ³n. BFGS realiza este proceso gradualmente mejorando la matriz hessiana de la funciÃ³n de coste. Dicha matriz se obtiene mediante evaluaciones del gradiente (o evaluaciones del gradiente aproximado) y se generaliza mediante el mÃ©todo de la secante [10].

Nota (MÃ©todo de la secante)

Se trata de un mÃ©todo utilizado para aproximar la derivada de una funciÃ³n, en algunos casos el cÃ¡lculo de la derivada es muy costoso por lo que este tipo de mÃ©todos resultan de gran utilidad. La expresiÃ³n utilizada para obtener el valor de la pendiente es la siguiente [14]:

\[ f^\prime(x_0) = \frac{f(x_1)-f(x_0)}{x_1-x_0} \]
