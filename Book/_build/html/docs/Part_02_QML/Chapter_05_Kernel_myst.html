5. Quantum Kernels#

En esta secci√≥n, se tratar√°n los m√©todos kernel. En primer lugar se presentar√°n los kernel cl√°sicos, para luego trasladar esta misma idea a la computaci√≥n cu√°ntica. A su vez, se presentar√°n algunas implementaciones de kernel cu√°ntico y se mostrar√° la matriz asociada a cada caso.

5.1. Introducci√≥n a los m√©todos kernel#

Los m√©todos kernel representan los datos en un n√∫mero de dimensiones mayor por lo que ayudan a encontrar nuevos patrones en los datos. Estas t√©cnicas son muy utilizadas en aprendizaje supervisado, ya que permiten transformar los problemas no lineales en lineales donde los datos son representandos en un espacio de mayor dimensi√≥n.

La expresi√≥n general de los kernel es \(k(\mathbf{x_{i}},\mathbf{x_{j}}) = \langle f(\mathbf{x_{i}}), f(\mathbf{x_{j}}) \rangle\) donde \(\mathbf{x_{i}}\) y \(\mathbf{x_{j}}\) corresponden a los datos de entradas de dimensi√≥n \(n\), \(f\) representa la funci√≥n que transforma los datos de entradas de dimensi√≥n \(n\) en un espacio de dimensi√≥n \(m\) y \(\langle \mathbf{a}, \mathbf{b} \rangle\) denota el producto interno. Considerando un conjunto de datos, la funci√≥n kernel se puede representar mediante una matriz, conocida como matriz kernel, \(K_{ij} = K(\mathbf{x_{i}},\mathbf{x_{j}})\) [5].

A continuaci√≥n,en la Fig. 9, se muestra un ejemplo visual para comprender la idea en la que se basan los kernel.

Este conjunto de datos (dataset en ingl√©s) cuenta con dos clases (azul y morada) y se aprecia una clara estructura en los datos de n dimensiones. Concretamente se distinguen dos circulos cada uno de ellos corresponde a una clase y uno est√° dentro del otro. Por lo tanto, en dos dimensiones no se puede separar las clases linealmente, en otras palabras, la frontera de decisi√≥n no ser√° lineal. Se trata de trasladar los datos a un espacio de dimensi√≥n mayor para as√≠ poder separar ambas clases. En este caso en particular se define la tercera dimensi√≥n como \(z = x^2+y^2\). En la Fig. 10 se muestra lo comentado:

Se observa que cuando los datos se representan en tres dimensiones existe un hiperplano que clasifica correctamente ambas clases. Proyectando el hiperplano de nuevo en las dimensiones originales la frontera de decisi√≥n es no lineal. Cabe destacar que esta manera de trabajar con la no linealidad en los modelos proporcionan la posibilidad de reconocer patrones en el espacio de datos original.

Nota

En algunas fuentes de informaci√≥n la expresi√≥n del kernel se expresa \(K(\mathbf{x_{i}},\mathbf{x_{j}}) = \phi(\mathbf{x_{j}})^T \phi(\mathbf{x_{i}})\), en este caso la funci√≥n que realiza el mapeo se denota mediante \(\phi()\). Esta funci√≥n tambi√©n se conoce como feature map [2].

Los m√©todos kernel son muy utilizados junto al algoritmo support vector machine (SVM) en conjuntos de datos no lineales, debido a la transformaci√≥n de los conjuntos de datos a lineales donde este algoritmo trata de encontrar el hiperplano \((\mathbf{w},b)\) que mejor separa las distintas clases. En este 4. Quantum Support Vector Machines (QSVM) se puede encontrar m√°s informaci√≥n a cerca de esta t√©cnica de clasificaci√≥n.

5.2. Introducci√≥n al kernel cu√°ntico#

Una vez se ha comentado la idea que persiguen los m√©todos kernel en computaci√≥n cl√°sica se aborda el funcionamiento de estas t√©cnicas en computaci√≥n cu√°ntica. Para visualizar la idea general y el funcionamiento de ambos tipos de kernel se adjunta la Fig. 11:

Como se aprecia en la Fig. 11 en ambos casos se trabaja con un feature map que se denota como \(\phi(\mathbf{x})\), se encarga de transformar los datos del espacio de datos original a un nuevo espacio de caracter√≠sticas. No obstante, aunque existen similitudes en lo que se refiere al concepto, tambi√©n se aprecian diferencias en la expresi√≥n de la funci√≥n kernel o que la funci√≥n kernel cu√°ntica adem√°s de llevar a cabo el cambio de dimensi√≥n, transforma los datos en estados cu√°nticos.

Tal y como se adelantaba en la comparaci√≥n representada en la Fig. 11, al tratar datos cl√°sicos medicante algoritmos de Quantum Machine Learning estos mismos deben encargarse de codificar los datos en estados cu√°nticos para posteriormente poder trabajar con ellos. Este proceso es equivalente a una funci√≥n feature map que asigna los valores a estados cu√°nticos, concretamente \(\phi(\mathbf{x})\) mapea el vector de caracter√≠sticas \(\mathbf{x}\) en el espacio de Hilbert \(\ket{\phi(\mathbf{x})}\bra{\phi(\mathbf{x})}\) [5]. Mediante la funci√≥n kernel se lleva a cabo la codificaci√≥n de los datos, conocida como data_encoding. El quantum kernel se puede interpretar como una medida de similitud, de forma que \(k(\mathbf{x_{i}},\mathbf{x_{j}})\) tomar√° un valor elevado cuando \(\mathbf{x_{i}}\) y \(\mathbf{x_{j}}\) sean cercanos [4].

La codificaci√≥n de los datos es una fase fundamental en QML y exiten varias t√©cnicas que abordan este reto, en este 2. Feature encoding se puede encontrar m√°s informaci√≥n a cerca de esta tem√°tica.

Entender los modelos cu√°nticos como m√©todos kernel significa que la expresividad, optimizaci√≥n y el comportamiento al generalizar los modelos est√°n determinados en gran medida por la estrategia de codificaci√≥n, la cual viene fijada por el m√©todo kernel [7]. Adem√°s significa que aunque el propio kernel puede explorar espacios de datos de dimensionalidad alta en el sistema cu√°ntico y los modelos cu√°nticos pueden entrenar y operar en subespacios de dimensiones m√°s bajas.

Al igual que en el caso cl√°sico, el puente entre el QML y los m√©todos kernel esta formado por la observaci√≥n de que los modelos cu√°nticos mapean los datos en una dimensi√≥n superior en la cual la medida define una frontera de decisi√≥n lineal.

5.3. Distintas implementaciones de kernel#

El entrenamiento basado en m√©todos kernel pasan por alto el procesado y la medici√≥n que implementan los circuitos variacionales, de manera que solo depender√° de la codificaci√≥n de los datos [3].

A continuaci√≥n se adjunta una implementeaci√≥n de distintos tipos de m√©todos kernel. Por un lado, se muestran dos ejemplos de Quantum Kernel Estimation (QKE). Por otro lado, se adjunta el c√≥digo para el denominado swap test. Para comprobar el correcto funcionamiento de estos ejemplos, se har√° uso del dataset iris, ya que se trata de un problema de clasificaci√≥n sencillo.

5.3.1. SWAP test#

Para implementar el kernel se preparan los estados \(\ket{\phi(\mathbf{x_{i}})},\ket{\phi(\mathbf{x_{j}})}\) en conjuntos de c√∫bits diferentes mediante rutinas de codificaci√≥n, \(U(\mathbf{x_{i}}), U(\mathbf{x_{j}})\). Generalmente este primer paso se suele llevar a cabo mediante rotaciones, en el ejemplo que se muestra a continuaci√≥n se trabaja con rotaciones en el eje Y. Tras codificar los datos cl√°sicos en estado cu√°nticos, se mide el solapamiento con una rutina que se denomina SWAP test [6].

En la Fig. 12 anterior se muestra el circuito necesario para implementar esta rutina y a continuaci√≥n se implementa esta rutina.

# Definimos el n√∫mero de qubits necesarios
nqubits=5

def circ_kernel_swap(x1, x2, nqubits):
    # Inicializamos el circuito con "nqubits"
    c= Circuit(nqubits=nqubits)
    
    # Generamos el Feature map mediante puertas RY
    c.add(gates.RY(q=1, theta=x1[0]))
    c.add(gates.RY(q=2, theta=x1[1]))
    c.add(gates.RY(q=3, theta=x2[0]))
    c.add(gates.RY(q=4, theta=x2[1]))
    
    # Kernel swap    
    c.add(gates.H(q=0))
    c.add(gates.SWAP(1,3).controlled_by(0))
    c.add(gates.SWAP(2,4).controlled_by(0))
    c.add(gates.H(q=0))
    
    # A√±adimos la medida en el qubit 0
    c.add(gates.M(0))
    
    return c


# Definimos el observable necesario para calcular el valor esperado
def hamiltonian():
    Obj = np.prod([ Z(0), I(1), I(2), I(3), I(4)])
    h = hamiltonians.SymbolicHamiltonian(Obj)
    return h

# Calculamos el valor esperado que produce el circuito
def expVal_swap_test(x1, x2, nqubits, nshots=10000):
    c = circ_kernel_swap(x1, x2, nqubits)
    h = hamiltonian()
    expected_value = h.expectation(c.execute().state())
    
    return expected_value

En la siguiente celda se comprueba el correcto funcionamiento de la funci√≥n kernel_swap, la comprobaci√≥n consiste en medir el solapamiento de un ejemplo con el mismo, por lo que el resultado debe ser 1.

Se puede observar de la matriz kernel lo siguiente:

  1. Cada fila / columna representa la amplitud de transici√≥n de una muestra, con todos las dem√°s muestras del conjunto de datos.

  2. La amplitud de transici√≥n de una muestra consigo mismo es 1, por lo que la matriz tiene una diagonal unitaria.

  3. La matriz es sim√©trica, la amplitud de transici√≥n de x \(\rightarrow\) y es igual y \(\rightarrow\) x

5.3.2. Quantum Kernel Estimator#

El Quantum Kernel Estimator, QKE, necesita la mitad de c√∫bits comparado con la versi√≥n de SWAP test y consiste en aplicar el operador unitario de codificaci√≥n \(\ket{U(\mathbf{x_{i}})}\) y su inverso en los mismos c√∫bits \(\bra{U^{\dagger}(\mathbf{x_{j}})}\) como se visualiza en la Fig. 8.

Por tanto, la f√≥rmula del kernel, K, se define como,

\[ K(\mathbf{x_{i}},\mathbf{x_{j}}) = |\braket{\phi(\mathbf{x_{j}})}{\phi(\mathbf{x_{i}})}|^2 = |\bra{0..0}U(\mathbf{x_{j}})^{\dagger}U(\mathbf{x_{i}})\ket{0..0}|^2\]

El QKE se puede implementar en dos versiones de circuitos que se expone a continuaci√≥n:

  1. proyector

  2. probabilidades

En las siguientes celdas se implementa el mismo kernel de dos formas distintas. En ambos casos la codificaci√≥n se lleva a cabo mediante angle embedding y luego se aplica el adjunto tal y como se comentaba. La diferencia viene a la hora de calcular el solapamiento, en el primer caso se utiliza el proyector y en el segundo se calcula la probabilidad de \(\ket{0}\).

Se hace uso de la codificaci√≥n angle embedding por lo que ser√°n necesarios tantos c√∫bits como car√°cteristicas tiene el dataset, se puede obtener m√°s informaci√≥n a cerca de esta codificaci√≥n en el siguiente 2. Feature encoding.

En primer lugar, se obtiene el n√∫mero de caracter√≠sticas de cada muestra en el dataset Iris, con el que se est√° trabajando en este ejemplo. Por ello, el n√∫mero de c√∫bits ser√° el n√∫mero de caracter√≠stica por muestra del conjunto de datos.

n_qubits = len(X_train[0])

5.3.2.2. Quantum Kernel Estimator (QKE) - probabilidades#

En este caso sin embargo, en lugar de utilizar el proyector se hace uso de la probabilidad del \(\ket{0}\) para calcular el solapamiento.

# kernel cu√°ntico
def distancia_circuit(x,y,n_qubits):
    
    # cuatro c√∫bits: cuatro caracter√≠sticas
    n_shots = 8000

    # Create an empty circuit
    circuit = Circuit(n_qubits)

    # Codificaci√≥n cu√°ntica en puertas RY 
    for pos in range(n_qubits):
        circuit.add(gates.RY(pos, y[pos]))
        circuit.add(gates.RY(pos, x[pos]).dagger())
        circuit.add(gates.M(pos))

    # ejecuci√≥n del circuito n veces (n_shots)
    result = circuit.execute(nshots=n_shots)

    # Devuelve las probabilidades anal√≠ticas
    return result.probabilities()
       
# Se devuelve el probabilidad del base computacional "0000"
kernel = lambda x1, x2, n_qubits: distancia_circuit(x1, x2, n_qubits)[0]

Se verifica que funciona correctamente el kernel cu√°ntico con un dato consigo mismo. El resultado deber√≠a ser 1.

#Se comprueba que el funcionamiento es adecuado.
kernel(X_train[0], X_train[0], nqubits) 

0.9999999999999996

Se analiza la matriz kernel

# Calculamos la matriz kernel al completo
matriz_prob = kernel_matrix(kernel,X_train,X_train,n_qubits)

sns.set(rc = {'figure.figsize':(12,8)})
sns.heatmap(matriz_prob, vmin = 0, vmax = 1, cmap=sns.color_palette("Blues", as_cmap=True));

Como se puede ver es indiferente calcular el solapamiento mediante proyectores o calculando la probabilidad de \(\ket{0}\), se puede observar que la matriz kernel es la misma siempre y cuando se aplique el mismo m√©todo de codificaci√≥n.

Nota (Anexo notaci√≥n)

Para que la comprensi√≥n de los notebooks sea mejor se ha unificado la notaci√≥n utilizada en los mismos. Para diferenciar un vector de un valor √∫nico se har√° uso de la negrita. De manera que \(\mathbf{x}\) corresponde a un vector y \(z\) ser√° una variable de una √∫nica componente.

Si se quiere hacer referencia a dos vectores distintos pero que pertenecen al mismo dataset se utilizar√° un sub√≠ndice, es decir, \(\mathbf{x_i}\) har√° referencia al i-√©simo vector del dataset. Si se quiere referenciar una caracter√≠stica concreta del vector se a√±adir√° un nuevo sub√≠ndice, de manera que \(\mathbf{x_{i_j}}\) har√° referencia a la j-√©sima variable del i-√©simo vector.
