5.1. IntroducciÃ³n a los mÃ©todos kernel#

Los mÃ©todos kernel representan los datos en un nÃºmero de dimensiones mayor por lo que ayudan a encontrar nuevos patrones en los datos. Este conjunto de datos (dataset en inglÃ©s) cuenta con dos clases (azul y morada) y se aprecia una clara estructura en los datos de n dimensiones. 5.3.1. SWAP test#

Para implementar el kernel se preparan los estados \(\ket{\phi(\mathbf{x_{i}})},\ket{\phi(\mathbf{x_{j}})}\) en conjuntos de cÃºbits diferentes mediante rutinas de codificaciÃ³n, \(U(\mathbf{x_{i}}), U(\mathbf{x_{j}})\). Generalmente este primer paso se suele llevar a cabo mediante rotaciones, en el ejemplo que se muestra a continuaciÃ³n se trabaja con rotaciones en el eje Y. Tras codificar los datos clÃ¡sicos en estado cuÃ¡nticos, se mide el solapamiento con una rutina que se denomina SWAP test [6].

En la Fig. 12 anterior se muestra el circuito necesario para implementar esta rutina y a continuaciÃ³n se implementa esta rutina.

# Definimos el nÃºmero de qubits necesarios
nqubits=5

def circ_kernel_swap(x1, x2, nqubits):
    # Inicializamos el circuito con "nqubits"
    c= Circuit(nqubits=nqubits)
    
    # Generamos el Feature map mediante puertas RY
    c.add(gates.RY(q=1, theta=x1[0]))
    c.add(gates.RY(q=2, theta=x1[1]))
    c.add(gates.RY(q=3, theta=x2[0]))
    c.add(gates.RY(q=4, theta=x2[1]))
    
    # Kernel swap    
    c.add(gates.H(q=0))
    c.add(gates.SWAP(1,3).controlled_by(0))
    c.add(gates.SWAP(2,4).controlled_by(0))
    c.add(gates.H(q=0))
    
    # AÃ±adimos la medida en el qubit 0
    c.add(gates.M(0))
    
    return c


# Definimos el observable necesario para calcular el valor esperado
def hamiltonian():
    Obj = np.prod([ Z(0), I(1), I(2), I(3), I(4)])
    h = hamiltonians.SymbolicHamiltonian(Obj)
    return h

# Calculamos el valor esperado que produce el circuito
def expVal_swap_test(x1, x2, nqubits, nshots=10000):
    c = circ_kernel_swap(x1, x2, nqubits)
    h = hamiltonian()
    expected_value = h.expectation(c.execute().state())
    
    return expected_value

5.3.2. Quantum Kernel Estimator#

El Quantum Kernel Estimator, QKE, necesita la mitad de cÃºbits comparado con la versiÃ³n de SWAP test y consiste en aplicar el operador unitario de codificaciÃ³n \(\ket{U(\mathbf{x_{i}})}\) y su inverso en los mismos cÃºbits \(\bra{U^{\dagger}(\mathbf{x_{j}})}\) como se visualiza en la Fig. 8.

Por tanto, la fÃ³rmula del kernel, K, se define como,

\[ K(\mathbf{x_{i}},\mathbf{x_{j}}) = |\braket{\phi(\mathbf{x_{j}})}{\phi(\mathbf{x_{i}})}|^2 = |\bra{0..0}U(\mathbf{x_{j}})^{\dagger}U(\mathbf{x_{i}})\ket{0..0}|^2\]

El QKE se puede implementar en dos versiones de circuitos que se expone a continuaciÃ³n:

def distancia_circuit(x,y,n_qubits):
    
    # cuatro cÃºbits: cuatro caracterÃ­sticas
    n_shots = 8000

    # Create an empty circuit
    circuit = Circuit(n_qubits)

    # CodificaciÃ³n cuÃ¡ntica en puertas RY
