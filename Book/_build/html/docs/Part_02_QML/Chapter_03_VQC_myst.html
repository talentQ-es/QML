3. Variational Quantum Classifier (VQC)#

Existen m√∫ltiples m√©todos para clasificar un conjunto de datos utilizando un ordenador cu√°ntico. Uno de ellos es el algoritmo conocido como Variational Quantum Classifier o VQC, que pertenece a la categor√≠a de algoritmos cu√°nticos variacionales (VQA).

El VQC es un algoritmo de aprendizaje supervisado que hace uso de un circuito cu√°ntico variacional. Este circuito, tambi√©n llamado circuito cu√°ntico parametrizado, tiene una estructura similar a la de una red neuronal cl√°sica. Siguiendo con esta analog√≠a, cada puerta cu√°ntica parametrizada equivale a un nodo en una red cl√°sica, y los par√°metros que gobiernan estas puertas a los pesos en una red.

Al igual que el aprendizaje autom√°tico cl√°sico, el algoritmo VQC tiene una fase de entrenamiento, en la que se proporcionan datos etiquetados, y una fase de validaci√≥n, en la que se clasifican nuevos datos con el modelo ya entrenado para as√≠ evaluar la calidad del modelo. Se trata de un algoritmo de clasificaci√≥n h√≠brido, ya que combina una componente cu√°ntica y una componente cl√°sica, tal y como se aprecia en la Fig. 6, que realiza un bucle de optimizaci√≥n de los par√°metros cl√°sicos del circuito cu√°ntico.

3.1. Variational Quantum Circuit#

Los circuitos cu√°nticos variacionales o parametrizados son circuitos que dependen de par√°metros configurables y que se optimizan mediante un coprocesador cl√°sico para realizar una tarea determinada.

En general, todo circuito variacional se divide en tres pasos:

  1. Preparaci√≥n de datos (State Preparation): El primer paso en cualquier proceso de aprendizaje autom√°tico cu√°ntico es trasladar los datos de tu problema cl√°sico a estados cu√°nticos dentro de tu espacio de Hilbert. Al hacer este proceso, es importante tener en cuenta que para clasificar correctamente los datos, debemos introducir no linealidad en el proceso, ya que el circuito cu√°ntico es lineal en el espacio de Hilbert. Esta no linealidad se puede introducir de diferentes formas, una de ellas es mediante un preprocesamiento cl√°sico de los datos o mediante las puertas cu√°nticas que transformar√°n estos datos cl√°sicos en caracter√≠sticas cu√°nticas de nuestro estado. Este proceso se conoce como codificaci√≥n de caracteristicas o feature encoding; es el encargado de traducir nuestro problema cl√°sico a un problema que nuestro circuito cu√°ntico pueda tratar. A este paso tambi√©n se le conoce como quantum embedding o data encoding. Para obtener m√°s informaci√≥n sobre estas t√©cnicas, puede consultar el siguiente 2. Feature encoding.

  2. Circuito parametrizado: El circuito dise√±ado para realizar una tarea concreta y est√° caracterizado por un operador unitario parametrizado \(U(\mathbf{x}, \theta)\), \(\mathbf{x}\) es la entrada del circuito (los datos codificados anteriormente) y \(\theta\) son los par√°metros variacionales. Este operador unitario est√° compuesto por diferentes capas de puertas cu√°nticas, la arquitectura para definir una capa puede cambiar seg√∫n el problema y debe adaptarse para cada caso. Tambi√©n existen variantes m√°s sofisticadas del VQC que combinan la arquitectura de capas con el mapeado de caracter√≠sticas, las cuales se discutir√°n m√°s adelante.

  3. Medida/Optimizaci√≥n: En esta etapa, se ejecuta el circuito y se mide repetidamente para obtener el valor esperado de un observable. Con este valor esperado, se intenta inferir la etiqueta que correspond√≠a al dato codificado. A partir de esta medida, se calcula una funci√≥n de coste y sus gradientes para poder optimizar los par√°metros que caracterizan el circuito y repetir el proceso. Con suficientes iteraciones y si el landscape de la funci√≥n de coste es navegable, los pesos de las puertas estar√°n optimizados para clasificar el conjunto de datos con el que se haya entrenado [7].

3.2. Data reuploading classifier#

La t√©cnica de data reuploading en clasificaci√≥n con circuitos cu√°nticos fue introducida en [30] y tiene la ventaja de introducir no linealidad en el sistema de manera nativa. El algoritmo consiste en aplicar t√©cnicas de feature encoding de manera iterativa al principio de cada capa. De esta manera, con un feature encoding sencillo y repitiendolo en cada layer se pueden resolver problemas de clasificaci√≥n no lineales con muy pocas capas.

def procesdata(data):
    data=np.array(data) #Pasamos los datos a una array
    Min=data.min(axis=0)
    Max=data.max(axis=0)
    data=(2*data-Max-Min)*np.pi/(Max-Min) #Centramos y acotamos entre -1 y 1
    return data

Definimos el ansatz de las capas del VQC que utilizaremos para construir el circuito. Como feature encoding utilizaremos el dense angle en dos qubits por lo que duplicaremos el feature encoding. De esta manera conseguimos de eficiente una buena no lienalidad.

def create_circuit(w,x, nqubits= 2,nlayers=nlayers):
    """
    Args:
        x: input del circuito
        w: par√°metros del circuito
        nqubits: n√∫mero de qubits
        nlayers: n√∫mero de capas
    Returns:
        instancia de circuito cu√°ntico para ejecutar
    """
    
    c= Circuit(nqubits=nqubits)

    c.add(gates.RY(q=0, theta = w[0]*x[0]))
    c.add(gates.RY(q=1, theta = w[1]*x[0]))
    
    c.add(gates.RZ(q=0, theta = w[2]*x[1]))
    c.add(gates.RZ(q=1, theta = w[3]*x[1]))
    
    for i in range(nlayers):
        j=i*4+3
        #j=7+i*4
        c.add(gates.RY(q=0, theta = w[j+1]))
        c.add(gates.RY(q=1, theta = w[j+2]))
     
        c.add(gates.CZ(0,1))
    
        c.add(gates.RY(q=0, theta = w[j+3]))
        c.add(gates.RY(q=1, theta = w[j+4]))
    
    # √öltima rotaci√≥n
    c.add(gates.RY(q=0, theta = w[j+5]))
    c.add(gates.RY(q=1, theta = w[j+6]))
    
    c.add(gates.M(0))
    
    return c

Ahora definimos las funciones que vamos a necesitar durante el entrenamiento. La funci√≥n ‚ÄúPredictions‚Äù computa los valores esperados de cada observable para cada dato que se ha dado de input al circuito.

#Definimos el observable que vamos a medir en este caso pauliz en el primer qubit
def hamiltonian(nqubits):
    if nqubits==1:
        Obj=Z(0)
    else:
        Obj = np.kron( Z(0), I(1))
    h = hamiltonians.SymbolicHamiltonian(Obj)
    return h

def Predictions(data, theta, nqubits, nshots=10000,R=False):
    """
    Args:
        data: input del circuito
        theta: par√°metros del circuito
        nqubits: n√∫mero de qubits
        nshots: n√∫mero de shots 
        R= par√°metro que controla si hacemos data reuploading
    Returns:
        numpy.array() con los valores esperados correspondientes al input
    """
    if R==True:
        c= create_circuitd(theta,data,nqubits)
        
    else:
        c =  create_circuit(theta,data, nqubits)
    h = hamiltonian(nqubits)
    expected_value = h.expectation(c.execute().state())
    return expected_value

Como funci√≥n de coste utilizamos el square loss defindo tal que: $\( \mathit{\text{Square loss}} = \frac {\sum_{i=1}^N (y_i - \tilde{y_i})^2}{N} \)$ Utilizaremos tambi√©n el accuracy o porcentaje de acierto para medir el rendimiento del modelo. Hay que tener en cuenta que los valores que tendremos como salida de ‚ÄúPredictions‚Äù ser√°n los valores esperados de un observable. Por consiguiente si queremos asignar una clase a cada valor podemos hacerlos dependiendo del signo, ya que el valor esperado est√° entre -1 y 1.

def sqloss_acc(labels, predictions):
    sqloss = 0
    acc = 0
    for label, prediction in zip(labels, predictions):    
        sqloss = sqloss + (label - prediction)**2
        if np.sign(label)==np.sign(prediction):
            acc += 1
    sqloss = sqloss / len(labels)
    acc = acc / len(labels)
    print('Loss: ', sqloss.numpy(), 'Acc: ', acc)
    return sqloss, acc

def Cost_function(theta, data, Y, nqubits, nshots=10000,R=False):

    predictions = [Predictions(x, theta,nqubits,nshots,R) for x in data]

    return sqloss_acc(Y, predictions)[0]

def create_circuitd(w,x, nqubits= 2,nlayers=nlayers):
    
    c= Circuit(nqubits=nqubits)
    for i in range(nlayers):
        c.add(gates.RY(q=0, theta = x[0]+w[2*i]))
        c.add(gates.RZ(q=0, theta = x[1]+w[2*i+1]))
    
    c.add(gates.M(0))
    
    return c

ANEXO NOTACI√ìN
