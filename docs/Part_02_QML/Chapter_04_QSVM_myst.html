4. Quantum Support Vector Machines (QSVM)#

En esta secci√≥n, se tratar√°n las m√°quinas de vectores de soporte o de soporte vectorial (en ingl√©s Support Vector Machines, SVM) tanto en computaci√≥n cl√°sica como en computaci√≥n cu√°ntica. En este √∫ltimo caso, la soluci√≥n es conocida como m√°quinas de vectores de soporte cu√°nticos (en su sigla en ingl√©s QSVM). A su vez, se presentar√°n algunas implementaciones de ambas computaciones.

4.1. Support Vector Machines (SVM)#

Las m√°quinas de vectores de soporte o de soporte vectorial son un conjunto de modelos de aprendizaje supervisado propuesto por Vladimir Vapnik .atl en 1964 y posteriormente fueron mejorados en diferentes versiones [9], [3]. Los SVMs se pueden aplicar en procesos de clasificaci√≥n, regresi√≥n y detecci√≥n de valores at√≠picos (en ingl√©s, outliers).

En clasificaci√≥n, es conocido como clasificador de vectores de soporte (SVC, en sus siglas en ingl√©s). El SVC construye el hiperplano √≥ptimo que maximiza la distancia entre dos categor√≠as del conjunto de datos para una mejor predicci√≥n. Es decir, busca el hiperplano con la mayor distancia de separaci√≥n entre las dos categor√≠as del conjunto de datos. Esta caracter√≠stica lo diferencia de los modelos de perceptr√≥n, que buscan el hiperplano que distingue las dos categor√≠as del conjunto de datos. Intuitivamente, una buena separaci√≥n se consigue mediante el hiperplano que tiene la mayor distancia al punto de datos de entrenamiento m√°s cercano de cualquier categor√≠a (llamado margen funcional), ya que, en general, cuanto mayor es el margen, menor es el error de generalizaci√≥n del clasificador [5].

En la Fig. 7 se visualizan tres posibles hiperplanos para separar las dos clases/categor√≠as del conjunto de datos, representadas por puntos negros y blancos. Los ejes \(x_{1}\) y \(x_{2}\) corresponden a las caracter√≠sticas de cada instancia en el conjunto de datos. El hiperplano H1 no separa adecuadamente las clases del conjunto de datos; el hiperplano H2 las separa con un margen peque√±o y, en cambio, el hiperplano H3 las distingue con la distancia √≥ptima. En consecuencia, se genera la frontera de decisi√≥n adecuada u √≥ptima para la predicci√≥n. En otras palabras, el SVM con el hiperplano H3 distinguir√° con mayor probabilidad la clase a la que pertenece un nuevo punto.

En general, un conjunto de datos no es linealmente separable y el SVC no encontrar√≠a un hiperplano √≥ptimo. En estos casos, se aplica el m√©todo del kernel, que permite el mapeo de los datos a otro espacio de caracter√≠sticas con mayor dimensi√≥n, donde ser√≠a m√°s sencillo encontrar el hiperplano √≥ptimo. Para m√°s detalle de los m√©todos kernel, visualice el notebook 5. Quantum Kernels.

4.2. Clasificador de vectores de soporte cu√°ntico#

El clasificador de vectores de soporte cu√°ntico (QSVC, en sus siglas en ingl√©s) es la aplicaci√≥n del SVC en computaci√≥n cu√°ntica, como se ilustra en la Fig. 8. Aprovecha la capacidad del SVC para trabajar con m√©todos kernel, los cuales pueden calcularse de manera eficiente en computaci√≥n cu√°ntica. De esta manera, se beneficia de la alta dimensionalidad del espacio de Hilbert [6], [8]. Para m√°s detalles de los m√©todos kernel, visualice el notebook 5. Quantum Kernels.

El QSVC corresponde en varios pasos como se visualiza en la Fig. 8:

  1. La generaci√≥n de un kernel cu√°ntico, \(K\):

    1. Mapeo de cada dato cl√°sico, \(\mathbf{x_{i}}\), en un estado cu√°ntico \(\vert \Phi{(\mathbf{x_{i}})} \rangle\). Esto se puede lograr con el circuito cu√°ntico \(\mathcal{U}_{\Phi{(\mathbf{x_{i}})}}\), \(\mathcal{U}_{\Phi{(\mathbf{x_{i}})}} \vert 0^{\otimes N} \rangle = \vert \Phi{(\mathbf{x_{i}})} \rangle\), donde el valor inicial de cada qubit es \(\ket{0}\) y \(N\) es el n√∫mero de caracter√≠sticas o qubits.

    2. Obtenci√≥n de la matriz del kernel, cada elemento de la matriz del kernel se calcula, \(K_{i,j} = |\braket{\Phi(\mathbf{x_{i}})}{\Phi(\mathbf{x_{j}})}|^2 = |\bra{0^{\otimes N}} \mathcal{U}^\dagger_{ \Phi(\mathbf{x_{i}})} \mathcal{U}_{ \Phi{(\mathbf{x_{j}})}} \ket{0^{\otimes N}}|^2 \) donde \(i,j = 0...M\) y \(M\) es el n√∫mero de vectores de caracter√≠sticas en el conjunto de datos. \(K_{i,j}\) se interpreta como una medida de similitud o de distancia entre estados cu√°nticos.

  2. Se aplica la matriz de kernel en el algoritmo de SVC cl√°sico para entrenar y obtener la predicci√≥n.

4.2.1. Ejemplo de QSVM en Qibo#

El ejemplo se basa en un tutorial de pennylane [6]. En este, se usar√° el SVM de la librer√≠a scikit-learn, sklearn. La interfaz permite proporcionar kernel adicional en dos formas:

  1. La funci√≥n que calcula la matriz del kernel.

    • (kernel= nombrefunci√≥n)

  2. La matriz del kernel calculada.

    • (kernel=‚Äúprecomputed‚Äù)

En el ejemplo, se aplicar√° la primera forma, facilitando la funci√≥n que calcula la matriz del kernel. A su vez, el famoso conjunto de datos, Iris, [4]. Este conjunto de datos, que data de 1936, consta de 150 muestras de cuatro caracter√≠sticas cada una y da lugar a un problema de clasificaci√≥n y se utilizar√° solo las primeras 100 muestras.

Se cargan las librer√≠as necesarias:

La codificaci√≥n de los datos cl√°sicos a estados cu√°nticos se llevar√° a cabo mediante la codificaci√≥n en √°ngulo, utilizando puertas de rotaci√≥n. Para obtener m√°s detalles sobre la codificaci√≥n cu√°ntica, consulte el notebook 2. Feature encoding. En el ejemplo, se aplica la puerta de rotaci√≥n en el eje Y para codificar los datos cl√°sicos en estados cu√°nticos. Esta puerta no introduce componentes imaginarias.

Dado que cada vector de caracter√≠sticas consta de cuatro caracter√≠sticas, se necesitar√°n cuatro qubits para codificar los estados cu√°nticos, por lo tanto, \(N = 4\).

Para estimar la distancia entre dos estados cu√°nticos, es decir, el producto escalar, se aplica el test invertido o quantum kernel estimation [6]. Para obtener m√°s detalles sobre los kernels, consulte el notebook 5. Quantum Kernels. En el ejemplo, el circuito cu√°ntico representa el c√°lculo de la distancia entre dos estados cu√°nticos como \(K_{i,j} = |\braket{\Phi(\mathbf{x_{i}})}{\Phi(\mathbf{x_{j}})}|^2 = |\bra{0^{\otimes N}} \mathcal{U}^\dagger_{ \Phi(\mathbf{x_{i}})} \mathcal{U}_{ \Phi{(\mathbf{x_{j}})}} \ket{0^{\otimes N}}|^2 \). El estado cu√°ntico resultante \(\ket{\phi}\) se mide con respecto al estado computacional \(\ket{0^{\otimes N}}\), y el resultado es la probabilidad \(|\braket {0^{\otimes N}} {\phi}|^2\) de observar la base computacional \(\ket{0^{\otimes N}}\) del estado cu√°ntico actual \(\ket{\phi}\). Este c√°lculo se basa en las reglas de Born [1].

Para obtener las probabilidades en la base computacional \(\ket{0^{\otimes N}}\) en una computadora cu√°ntica, el circuito cu√°ntico debe ser ejecutado varias veces o shots. Seg√∫n el art√≠culo Supervised learning with quantum-enhanced feature spaces [6], el circuito deber√≠a ejecutarse 50000 veces. En el ejemplo, se usa 8000 por limitaci√≥n del computador cu√°ntico.

# kernel cu√°ntico
def distancia_circuit(x,y):
    
    # cuatro c√∫bits: cuatro caracter√≠sticas
    n_qubits = len(X_train[0]) 
    n_shots = 8000

    # Create an empty circuit
    circuit = Circuit(n_qubits)

    # Codificaci√≥n cu√°ntica en puertas RY 
    for pos in range(n_qubits):
        circuit.add(gates.RY(pos, y[pos]))
        circuit.add(gates.RY(pos, x[pos]).dagger())
        circuit.add(gates.M(pos))

    # ejecuci√≥n del circuito n veces (n_shots)
    result = circuit.execute(nshots=n_shots)
    # Devuelve las probabilidades anal√≠ticas
    return result.probabilities()
       
# Se devuelve el probabilidad del base computacional "0000"
kernel = lambda x1, x2: distancia_circuit(x1, x2)[0]

Se verifica que el kernel cu√°ntico funciona correctamente con el vector de caracter√≠sticas \(\mathbf{x}\) consigo mismo. El resultado deber√≠a ser 1, es decir, la probabilidad del estado inicial \(\ket{\phi}{\text{inicial}} = \ket{0^N}\) es igual a la probabilidad del estado final \(\ket{\phi}{\text{final}} = \ket{0^N}\).

x = np.array([np.pi/2,np.pi/2,np.pi/2,np.pi/2])
kernel(x,x)

Se define una funci√≥n que calcula una matriz de kernel para los vectores de caracter√≠sticas en dos conjuntos de datos diferentes \(A, B\). Si \(A=B\), la matriz del kernel es conocida como la matriz de Gram [2].

def kernel_matrix(A, B):
    return np.array([[kernel(a, b) for b in B] for a in A])

Nota (Anexo notaci√≥n)

Para que la comprensi√≥n de los notebooks sea mejor se ha unificado la notaci√≥n utilizada en los mismos. Para diferenciar un vector de un valor √∫nico se har√° uso de la negrita. De manera que \(\mathbf{x}\) corresponde a un vector y \(z\) ser√° una variable de una √∫nica componente.
