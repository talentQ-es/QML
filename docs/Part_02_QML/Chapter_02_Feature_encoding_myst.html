2. Feature encoding#

Esta secci√≥n se centra en el proceso de codificaci√≥n de caracteristicas o feature encoding, se encarga de codificar datos cl√°sicos en estados cu√°nticos. Es un parte muy delicada en procesamiento de la informaci√≥n cu√°ntica debido a que te dar√° el mapa que proyectar√° tus datos cl√°sicos en el espacio de hilbert.

Nota

Dependiendo de c√≥mo est√© proyectado este conjunto de datos en el espacio de Hilbert, el tratamiento que se le tendr√° que hacer para extraer informaci√≥n ser√° muy simple o, si no se hace adecuadamente, imposible.

Se empieza con un conjunto de datos \(\mathbf{x}\) y se quiere codificar en un estado \(\ket{\psi_x}\) con el que trabajar, para ello se construye un circuito parametrizado que generar√° una unitaria \(\mathcal{U}(\mathbf{x})\) tal que \(\ket{\psi_x}=\mathcal{U}(\mathbf{x})\ket{0}\). Dependiendo del tipo de feature encoding que se utilice, la unitaria \(\mathcal{U}(x)\) tendr√° diferentes caracter√≠sticas.

Se define un conjunto de datos (dataset) de entrada \(D\) de dimensi√≥n \(M\) x \(N\), es decir, se cuenta con un dataset con M instancias y N variables. Este dataset se puede expresar como \(D = {\mathbf{x}_{(1)}, ..., \mathbf{x}_{(m)}, ..., \mathbf{x}_{(M)}}\) donde cada \(\mathbf{x}_{(m)}\) es un vector de dimensi√≥n N.

Hay multiples t√©cnicas de feature encoding, a continuaci√≥n se explicar√°n las m√°s b√°sicas y se dar√° un ejemplo de c√≥mo implementar cada una. Para m√°s detalles consultar[24],[36].

2.1. Basis encoding#

La t√©cnica Basis encoding, tambi√©n conocida como Basis Embedding, asocia cada elemento del dataset a un estado de la base computacional de un sistema de qubits. Es por eso que los datos cl√°sicos deben estar representados por una cadena de bits de longitud equivalente al n√∫mero de qubits del circuito. Esto da una correspondencia uno a uno entre elementos del dataset y vectores de la base computacional del espacio de Hilbert.

Se considera el conjunto de datos \(D\), definido anteriormente. Asumimos por simplicidad que podemos codificar las \(N\) caracter√≠sticas con un n√∫mero binario de longitud \(N\), tal que \(\mathbf{x}_m = (b_1,...,b_N)\) con \(b_i\in { 0,1}\) para \(i=1,...,N\). De esta manera, cada elemento del conjunto de datos \(\mathbf{x}_{m}\) se puede representar como \(\ket{\mathbf{x}_{m}}\). Extendiendo la expresi√≥n anterior a todo el dataset, obtenemos la representaci√≥n del conjunto de datos al completo tal que:

\[\ket{D} = \frac{1}{\sqrt M} \sum^M_{m=1} \ket{\mathbf{x}_{m}}\]

Por ejemplo, si consideramos que el dataset \( D=\{1,3,5,7\} \) los datos en binario corresponden a \( D=\{001,011,101,111\} \). En este caso, el estado correspondiente ser√≠a:

\[\ket{D} = \frac{1}{\sqrt{4}}(\ket{001}+\ket{011}+\ket{101}+\ket{111})\]

Que como vector en la base computacional corresponde a \(\vec{x}=\frac{1}{2}(0,1,0,1,0,1,0,1)\).

2.2. Amplitude encoding#

La t√©cnica Amplitude encoding o amplitude embedding, codifica los datos cl√°sicos como las amplitudes del vector de un estado cu√°ntico. En este caso, un elemento \(\mathbf{x_m}=(b_0,...b_N)\) del conjunto de datos \(D\) con \(N\) caracter√≠sticas se codifica en un sistema de \(2^n=N\) qubits tal que:

\[ \ket{\psi_x} = \sum^N_{i=1} b_i\ket{i}\]

Donde \(b_i\) es el i-√©simo elemento del vector \(\mathbf{x_m}\) y \(\ket{i}\) es el i-√©simo estado de la base computacional. A diferencia de la t√©cnica basis encoding, \(x_i\) puede tomar valores de distintos tipos, como integer (valores enteros) o float (valores de coma flotante).

Como ejemplo, si tratamos de codificarel vector \(\mathbf{x}=(1,3,5,7)\) utilizando esta t√©cnica obtendremos:

\[\ket{\psi_x} = \frac{1}{2\sqrt{21}} [1\ket{00}+3\ket{01}+5\ket{10}+7\ket{11}]\]

Nota

Hay que tener en cuenta que todos los vectores cu√°nticos deben estar siempre normalizados, \(\bra{\psi_x}\ket{\psi_x} = 1\)

2.3. Angle encoding#

Angle encoding es la tecnica de codificaci√≥n m√°s f√°cil de implementar debido a que utiliza las puertas parametrizadas m√°s basicas para ello. Las N caracter√≠sticas de los elementos del dataset se codifican como √°ngulos de rotaci√≥n de N c√∫bits. Esta metodolog√≠a codifica N caracter√≠sticas en un √°ngulos de rotaci√≥n de N qubits. Dichas rotaciones pueden llevarse a cabo en cualquier eje, tanto en el \(X\), como en el \(Y\) o en el \(Z\). Mas explicitamente si queremos codificar el vector del conjuto de datos \(\mathbf{x_m}=(x_1,...,x_N)\) con rotaciones en el eje \(X\) el estado cu√°ntico resultante ser√≠a:

\[ \ket {\mathbf{x}} = \bigotimes^N_{i=1} \cos(x_i)\ket 0 + \sin(x_i)\ket 1 \]

Por ejemplo, si se trata de codificar el vector \(\mathbf{x}=(1,3,5,7)\) con este tipo de encoding, se necesitar√°n cuatro c√∫bits y quedar√° como sigue:

\[ \ket{\mathbf{x}} = (\cos(1)\ket{0} + \sin(1)\ket{1} ) \otimes (\cos(3)\ket{0} + \sin(3)\ket{1}) \otimes (\cos(5)\ket{0} + \sin(5)\ket{1}) \otimes (\cos(7)\ket{0} + \sin(7)\ket{1} ) \]
num_q=4

def Angle(x,nqubits):
    c= Circuit(nqubits=nqubits)
    
    for i in range(nqubits):
        c.add(gates.RX(q=i,theta=x[i]))
    return c

x = np.array([1,3,5,7])

c_an=Angle(x,num_q)

2.2.4. Dense angle encoding#

Esta t√©cnica, tal como su nombre indica, es una versi√≥n m√°s sofisticada de la codificaci√≥n anterior. Es capaz de codificar dos caracter√≠sticas por cada qubit, haciendo uso de fases relativas. En este caso, la instancia \(\mathbf{x}=(x_1,...,x_N)\) se codifica como sigue:

\[ \ket{x} = \bigotimes^{N/2}_{i=1} \cos(x_{2i-1})\ket{0} + e^{ix_{2i}}\sin(x_{2i-1})\ket{1} \]

Nota

Angle encoding necesita \(num \, qubits = \text{dim}(\mathbf{x})\), mientras que dense angle requiere \(num \, qubits = \text{dim}(\mathbf{x})/2\). Aun as√≠, hay aplicaciones donde la simplicidad de angle encoding es preferible.

Por ejemplo, si se trata de codificar el vector \(\mathbf{x}=(1,3,5,7)\) con este tipo de encoding, se necesitar√°n dos c√∫bits y quedar√° como sigue:

\[ \ket{\mathbf{x}} = (\cos(1)\ket{0} + e^{i3}\sin(1)\ket{1} ) \otimes (\cos(5)\ket{0} + e^{i7}\sin(5)\ket{1}) \]
num_q=2

def denseAngle(x,nqubits):
    c= Circuit(nqubits=nqubits)
    
    for i in range(nqubits):
        c.add(gates.RX(q=i,theta=x[i]))
        c.add(gates.RZ(q=i,theta=x[i+1]))
    return c

x = np.array([0,1,2,3])

c_dan=denseAngle(x,num_q)

print("x               : ", x)
print("amplitude vector: ", np.array(c_dan.execute().state()))

2.3. C√≥mo escoger Feature Encoding#

Cuando se trata con circuitos variacionales, la decisi√≥n de qu√© feature encoding utilizar es crucial. Los diferentes feature encoding presentan diferentes ventajas e inconvenientes dependiendo del problema que queramos resolver. Las t√©cnicas mencionadas en este notebook se pueden separar en dos tipos:

  • Basis encoding donde se trabaja con los elementos de la base computacional como inputs (entradas).

  • El resto de codificadores que trabajan con las amplitudes del vector cu√°ntico.

La t√©cnica de basis encoding presenta la capacidad de calcular operaciones no lineales de forma natural, a cambio es el codificador que presenta mayor n√∫mero de problemas:

  • Es el m√°s susceptible a errores.

  • Escala muy mal con el n√∫mero de c√∫bits, ya que de esto depende la precisi√≥n de los feature que se quiere codificar.

  • Es el m√©todo que presenta mayor dificultad a la hora de entrenar la red neuronal y tiene tendencia a presentar barren plateaus (gradientes que tienden a cero).

El resto de m√©todos basados en codificar en las amplitudes del vector cu√°ntico no presentan los inconvenientes enumerados anteriormente y son los m√©todos preferidos para desarrollar algoritmos en la era del NISQ (Noisy Intermediate-Scale Quantum). A√∫n as√≠, presentan el inconveniente de que hay que escoger sabiamente c√≥mo introducir la no linealidad necesaria para cada problema. La forma m√°s gen√©rica es utilizar Amplitude encoding y aplicar una funci√≥n \(f(x)\) a tus datos para obtener correlaciones entre ellos y as√≠ poder resolver problemas no lineales. Este m√©todo, que es a primera vista sencillo, presenta el problema de que la amplitud del circuito puede ser muy grande dependiendo de \(f(x)\) y adem√°s, escoger \(f(x)\) puede ser un proceso muy arbitrario. Por otra parte, otros m√©todos de codificaci√≥n m√°s sencillos pueden proporcionar directamente los elementos de no linealidad necesarios para resolver el problema.

Para finalizar, existen otras t√©cnicas m√°s sofisticadas para introducir datos cl√°sicos en un circuito variacional, una de ellas es conocida como data re-uploading [30] el cual se cubrira en el siguiente notebook.
